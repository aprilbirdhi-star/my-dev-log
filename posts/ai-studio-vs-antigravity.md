# Google AI Studio vs 안티그래비티 (Antigravity) 차이점 & 주요 AI 용어 정리

Google AI Studio와 안티그래비티(Antigravity)는 AI를 활용해 무언가를 만든다는 점에서는 비슷해 보이지만, **타겟 독자, 작업의 목적, 그리고 구현하는 방식**에서 아주 명확한 차이가 있습니다.

쉽게 비유하자면, Google AI Studio는 '고성능 엔진과 부품을 제공하는 전문 정비소'이고, 안티그래비티는 '나만의 맞춤형 자율주행 차량을 조립하는 프레임워크'라고 볼 수 있습니다.

두 플랫폼의 핵심적인 차이점을 직관적으로 비교해 드릴게요.

---

## 1. 핵심 목적 및 타겟층의 차이

### 🛠️ Google AI Studio

* **목적:** 구글의 최신 거대언어모델(Gemini 등)을 직접 테스트하고, 그 모델을 구동할 수 있는 **API 키를 발급받거나 프롬프트를 정교하게 엔지니어링**하는 개발자 중심의 실험실입니다.
* **특징:** 여기서 '앱을 만든다'는 것은 완전한 형태의 독립된 앱(UI가 있는 프로그램)을 완성한다기보다, "내 앱에 들어갈 AI의 두뇌(기능)를 설계하고 API 형태로 포장한다"에 가깝습니다. 즉, 결과물을 가지고 다른 개발 환경(VS Code, 파이썬 등)으로 가져가서 진짜 앱을 빌드해야 합니다.

### 🚀 안티그래비티 (Antigravity)

* **목적:** 사용자가 복잡한 코딩 없이도 AI 에이전트, 워크플로우 자동화, 나만의 맞춤형 콘텐츠 생성 도구를 **시각적으로 조립하고 곧바로 실행**할 수 있도록 돕는 플랫폼입니다.
* **특징:** LLM(Gemini, Claude 등)의 API를 연동하여, 실제 업무나 창작에 바로 쓸 수 있는 '시스템'이나 '자동화 파이프라인'을 구축하는 데 초점이 맞춰져 있습니다. 모델 그 자체를 커스텀하기보다, 여러 모델과 도구들을 조합해 하나의 완성된 서비스를 만드는 것에 가깝습니다.

---

## 2. 기능 및 작업 방식 비교

| 비교 항목 | Google AI Studio | 안티그래비티 (Antigravity) |
| --- | --- | --- |
| **주요 역할** | Gemini 모델 프롬프트 최적화, API 키 관리 | AI 에이전트 빌드, 워크플로우 및 자동화 설계 |
| **사용 모델** | 오직 **Google Gemini** 제품군만 지원 | Gemini, Claude 등 **다양한 오픈/폐쇄형 모델** 연동 가능 |
| **구현 방식** | 시스템 지시문(System Instruction), 퓨샷 라이팅(Few-shot) 테스트 | 노드 기반 시각적 연결, MCP(Model Context Protocol) 등 외부 도구 결합 |
| **최종 결과물** | 다른 프로그램에 삽입할 **API 코드, 프롬프트 JSON** | 곧바로 작동하는 **AI 에이전트, 자동화 워크플로우** |
| **난이도** | 개발 지식이 약간 필요 (코드 스니펫 제공) | 직관적인 UI 중심 (프롬프트 엔지니어링 및 아키텍처 이해 필요) |

---

## 3. 요약: 언제 무엇을 써야 할까?

### 💡 Google AI Studio를 써야 할 때
* Gemini 모델의 순수한 성능(텍스트, 이미지, 오디오 멀티모달 인식 등)을 정교하게 테스트하고 싶을 때
* 내가 직접 코딩해서 만들 웹/앱 서비스에 넣을 **최적의 프롬프트와 API 키**가 필요할 때
* 구글의 시스템 지시문(System Instruction) 설정을 완벽하게 빌드하고 싶을 때

### 💡 안티그래비티를 써야 할 때
* 코드 한 줄 안 짜고도 여러 AI 모델을 조합해 **복잡한 창작/업무 자동화 에이전트**를 만들고 싶을 때
* 로컬 환경(LM Studio, Ollama 등)이나 다양한 외부 API, MCP 서버 등을 연결해 **확장성 있는 나만의 AI 허브**를 구축하고 싶을 때
* 프롬프트 설정을 넘어, '데이터 입력 ➡️ AI 분석 ➡️ 결과물 생성 ➡️ 파일 저장' 등의 전체 프로세스(워크플로우)를 구현하고 싶을 때

> 즉, AI의 '두뇌와 엔진' 자체를 정교하게 튜닝하려면 **Google AI Studio**가 유리하고, 그 두뇌들을 조립해서 실무에 바로 쓸 '로봇(에이전트나 시스템)'을 만들려면 **안티그래비티**가 훨씬 직관적이고 강력합니다.

## 4. 초보자를 위한 핵심 기술 용어 해설

낯선 기술 용어들이 섞여 있어서 조금 어렵게 느껴지셨을 수 있습니다. 아주 쉽게, 일상적인 비유를 들어 하나씩 설명해 드릴게요.

### ① UI (User Interface, 사용자 인터페이스)란?

UI(유이)는 한마디로 **"인간과 컴퓨터가 만나는 화면(소통 창구)"**을 말합니다.

우리가 스마트폰이나 컴퓨터를 쓸 때, 복잡한 컴퓨터 코드(0과 1)를 직접 치지 않죠? 대신 화면에 있는 예쁜 **버튼, 메뉴, 아이콘, 입력창**을 마우스로 누르거나 손가락으로 터치합니다. 이처럼 사용자가 기술을 쉽게 쓸 수 있도록 시각적으로 만들어 놓은 모든 요소를 UI라고 부릅니다.

* **텍스트 중심 UI (CLI/CUI):** 과거 검은 화면에 흰 글자만 나오던 프롬프트 창 (메모장이나 터미널)
* **직관적인 UI (GUI):** 안티그래비티처럼 마우스로 상자를 드래그하고, 선으로 연결하고, 버튼을 누르면 직관적으로 작동하는 화면

### ② 프롬프트 엔지니어링 (Prompt Engineering)

프롬프트 엔지니어링은 **"AI에게 원하는 답을 얻기 위해, 질문이나 지시(프롬프트)를 정교하게 디자인하는 기술"**입니다.

AI는 똑똑하지만 마음을 읽지 못하기 때문에, 우리가 어떻게 말하느냐에 따라 결과물의 퀄리티가 하늘과 땅 차이로 달라집니다. 단순히 "글 써줘"라고 하는 대신, **역할, 배경 상황, 제약 조건, 출력 형식**을 명확히 정해주는 과정이 바로 프롬프트 엔지니어링입니다.

> **💡 예시**
> * **일반 질문:** "강아지 주인공으로 이야기 써줘."
> * **프롬프트 엔지니어링:** "너는 픽사 스타일의 동화 작가야. 이름은 '티티'이고, 하얗고 조그만 말티즈 강아지가 주인공이야. 우주를 여행하며 별을 수집하는 모험 이야기를 3인칭 시점으로, 따뜻한 톤앤매너로 3문단 이내로 작성해줘."

### ③ 아키텍처(Architecture)의 이해

아키텍처는 원래 건축학에서 '설계도'를 뜻하는 말입니다. AI나 소프트웨어에서 아키텍처란 **"여러 가지 기능과 데이터가 어떤 순서로, 어떻게 연결되어 움직이는지 그려낸 '전체적인 시스템 구조 설계도'"**를 의미합니다.

집을 지을 때 거실, 안방, 화장실을 어디에 배치하고 배수관을 어떻게 연결할지 설계하는 것처럼, AI 앱을 만들 때도 구조를 짜야 합니다.

안티그래비티 같은 플랫폼에서 "아키텍처 이해가 필요하다"는 말은, 코딩은 안 하더라도 **데이터가 흘러가는 길을 설계할 줄 알아야 한다**는 뜻입니다.

> **🛠️ AI 워크플로우 아키텍처 예시 (내가 만드는 설계도)**
> 1. **입력:** 사용자가 '오늘 날씨'를 입력한다.
> 2. **1단계 (날씨 API):** AI가 오늘 날씨 데이터를 외부에서 받아온다.
> 3. **2단계 (LLM/Gemini):** 받아온 날씨 데이터를 AI에게 주고 "이 날씨에 어울리는 감성적인 시 한 편 써줘"라고 명령한다.
> 4. **3단계 (이미지 생성):** AI가 쓴 시의 분위기에 맞는 이미지를 AI가 자동으로 그리게 한다.
> 5. **출력:** 최종 완성된 시와 이미지를 화면에 보여준다.

---

### ✍️ 요약하자면,
안티그래비티 같은 도구는 복잡한 코딩(C언어, 파이썬 등)은 몰라도 되도록 시각적인 화면(UI)을 잘 제공해 주지만, 내가 원하는 결과물을 완성도 있게 만들려면 AI에게 명령을 잘 내리는 방법(프롬프트 엔지니어링)과 기능들을 어떤 순서로 연결할지 짜는 머릿속 설계도(아키텍처)가 필요하다는 의미입니다.
