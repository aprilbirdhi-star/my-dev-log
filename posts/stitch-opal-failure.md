# 실패노트: Google Stitch 무한 로딩 오류와 Opal 블랙유머 웹툰 재생성 실패기

> **"오늘 기획하는 것 모두 망하고 있어 일단 좀 쉬어야겠다."**
> 오늘은 아쉽게도 시도했던 모든 기획과 개발이 실패로 끝난 '망한 날'이다. 하지만 실패 속에서도 배운 도구들의 생리와 원인을 생생히 기록해 둔다.

## 1. Google Stitch 모바일 앱 빌드 및 연동 실패

모바일 앱의 UI 디자인과 흐름을 설계하기 위해 구글 스티치(Stitch)를 활용했다. 스티치에서 완성도 높은 앱 디자인을 구축한 뒤, API Key를 발급받아 Antigravity(안티) 개발 환경으로 이식하여 앱을 실제 코드로 구현하고자 시도했다.

* **시도한 과정**:
  1. Stitch 웹 콘솔에서 모바일 앱 스크린 및 UI 레이아웃 설계.
  2. 스티치 API Key 복사 후 개발 프로젝트 환경변수 및 빌드 파일(`build.js` 등)에 적용.
  3. Antigravity 에이전트를 통해 레이아웃 코드를 빌드하고 시뮬레이터 또는 테스트 모바일 앱 구동 시도.

* **발생한 문제**:
  * 빌드 및 실행 도중 **원인을 알 수 없는 무한 로딩(Infinite Loading)** 현상이 발생했다.
  * 네트워크 요청 로그, 콘솔 출력, API Key 권한 설정을 여러 차례 점검하고 코드를 지속적으로 수정했으나 로딩 바가 멈추지 않고 화면이 멈추는 에러가 해결되지 않았다.
  * 결국 모바일 테스트 앱 화면을 정상적으로 띄우지 못하고 실패로 마무리했다.

---

## 2. 구글 오팔(Google Opal)을 통한 블랙유머 웹툰 재생성 실패

어제 겪었던 캐릭터 이미지 일관성(Character Consistency) 한계와 생성량 초과 이슈를 극복하기 위해, 오늘은 구글 오팔(Opal) 환경에서 블랙유머 웹툰 컷을 다시 생성해 보고자 했다.

* **발생한 문제**:
  * 프롬프트를 세부 조정하고 생성 파라미터를 다르게 시도했으나, 오팔 시스템에서도 원하는 느낌의 고유 캐릭터 이미지 재생성에 실패했다.
  * 이미지 스타일의 정합성이 흐트러지거나, 웹툰 특유의 연출 의도와 다른 결과물이 반복 생성되어 결국 결과물을 확보하지 못했다.

---

## 3. 번외: 그래도 오늘의 수확, 필로그(Philog) 공식 로고 디자인!

오늘 시도한 모바일 앱 빌드와 웹툰 생성은 장벽에 부딪혔지만, 브랜드 정체성을 더욱 확고히 다지기 위해 **새로운 필로그(Philog.today) 공식 로고**를 디자인하고 제작하는 데 성공했다.

* **로고 디자인 콘셉트**:
  * 기록을 상징하는 필로그(Philog)의 톤앤매너를 현대적으로 녹여냈으며, 웹사이트와 브라우저 파비콘(Favicon)에 즉각 반영했다.
  * 새로운 로고 덕분에 포트폴리오 사이트가 한층 정돈되고 세련된 느낌을 가지게 되었다.

---

## 4. 실패의 날, 하지만 도구들과 한 걸음 더 가까워진 하루

비록 오늘의 결과물은 전부 '실패'라는 도장으로 끝났지만, 얻은 것은 분명하다.

> "실패는 도구와 친해지는 가장 빠른 지름길이다."

구글 스티치의 API 연동 방식, 안티 에이전트의 코드 파싱 방식, 그리고 구글 오팔의 이미지 제어 능력에 대해 직접 부딪히며 구조적인 흐름을 이해할 수 있었다. 에러가 왜 발생했는지, 무한 로딩의 병목 구간이 어디인지 분석하는 과정 속에서 개발 툴들의 생리를 더 깊게 이해하게 되었다.

내일은 오늘 발견한 무한 로딩 현상의 상세 에러 로그를 캡처하고, 스티치 API 문서와 라이브러리 스펙을 다시 분석하여 근본적인 실패 원인을 하나씩 제거해 나갈 계획이다.

**오늘도 밭에 실패라는 비료를 뿌렸다. 더 단단한 싹을 틔우기 위한 밑거름이 될 것이다.**
