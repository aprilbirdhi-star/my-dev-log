# 💡 C드라이브 용량 절약과 GitHub 중심의 멀티 디바이스 개발 환경 구축

안녕하세요! 이루다입니다. 

오늘 작업 중 개발 도구의 쿼터 초과 문제와 로컬 컴퓨터의 C드라이브 용량 부족 우려가 겹쳐, 코드 보관 방식을 어떻게 해야 최적일지에 대한 의문이 생겼습니다. 

보통 개인 프로젝트를 구글 드라이브(Google Drive)에 올려두고 다운로드해 쓰는 것을 가장 먼저 떠올리기 쉽지만, 개발의 세계에서는 **GitHub**가 가장 완벽한 해답이 됩니다.

---

## 💻 왜 구글 드라이브 대신 GitHub인가요?

### 1. 0.001%도 아쉬운 디스크 용량 세이브 (약 2MB 내외)
이 정적 포트폴리오 웹사이트 폴더(`philog.today`)의 용량은 이미지 파일 몇 장을 포함해도 **약 2MB 미만**입니다. 스마트폰 사진 한 장 용량보다 작으므로 C드라이브 공간은 걱정하지 않으셔도 되며, 원격 클라우드에 온전히 업로드되어 있기 때문에 필요할 때 로컬 폴더를 언제든지 자유롭게 삭제해도 안전합니다.

### 2. 구글 드라이브의 치명적 충돌 문제 회방
구글 드라이브와 같은 웹 동기화 서비스는 파일을 실시간으로 서버에 올립니다. 이때 코드 작업 중 생성되는 숨겨진 임시 파일이나 빌드용 파일들이 잠기거나 꼬이면서 **소스 코드가 유실되거나 깃 시스템 폴더가 손상되는 CORS/잠금 오류**를 유발할 수 있습니다. 반면 Git은 수동으로 완벽한 커밋 지점만 커맨드로 푸시하므로 매우 안전합니다.

### 3. Vercel 배포 자동화
현재 `philog.today` 도메인은 GitHub 저장소와 Vercel 플랫폼이 유기적으로 엮여 있습니다. 코드를 깃에 올리는 순간, Vercel이 이를 감지하여 **자동으로 새 코드를 빌드하고 무중단 배포**를 수행합니다. 구글 드라이브에 파일을 저장하면 이 모든 배포 자동화 체인이 끊어집니다.

## 🚀 새로운 기기에서 언제든 이어서 작업하는 핵심 방법

나중에 새 노트북이나 컴퓨터에서 내 웹사이트를 다운받아 다시 세팅하고 싶다면, 구글 드라이브에 갈 필요 없이 아래 터미널 명령어 한 줄만 치면 됩니다.

```bash
git clone https://github.com/aprilbirdhi-star/philog.today.git
```

또한, 급하게 외부 PC나 태블릿(iPad) 환경에서 수정해야 할 때는 설치할 필요 없이 웹 브라우저로 자신의 깃허브 저장소에 접속하여 **키보드의 마침표(`.`) 키**를 누릅니다. 그러면 브라우저 안에 클라우드 전용 웹 VS Code가 즉시 켜져 간편하게 편집하고 푸시할 수 있습니다.
